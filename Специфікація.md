# SPECIFICATION - po-tools

## Огляд (Overview)
`po-tools` — це набір інструментів командного рядка для підвищення продуктивності при роботі з файлами GNU Gettext PO.
Він надає можливості для автоматизованого перекладу (за допомогою AI), рецензування, порідняння, злиття та валідації перекладів.
Важливою особливістю po-tools є підтримка msgid_plural.

## Архітектура (Architecture)
Проект реалізований на мові Rust.
- **Точка входу**: `src/main.rs`. Обробляє аргументи командного рядка вручну (без використання `clap` або `structopt` для основного парсингу), ініціалізує `Parser` та диспетчеризує виконання до відповідних модулів команд.
- **Модульність**: Кожна команда винесена в окремий модуль (`command_*.rs`), що забезпечує чітке розділення логіки.
- **Парсер**: Модуль `parser.rs` (struct `Parser`) відповідає за читання та обробку PO файлів.

## Основні функції (Key Features)

### Реалізовані (або частково реалізовані) команди:
- **Маніпуляція файлами**:
    - `merge`: Злиття двох файлів (перезапис повідомлень).
    - `sort`: Сортування повідомлень у лексичному порядку.
    - `erase`: Очищення перекладів.
- **Аналіз та порівняння**:
    - `diff`: Порівняння файлів за `msgid`.
    - `diffstr`: Порівняння файлів за `msgstr`.
    - `added` / `removed` / `same`: Пошук нових, видалених або однакових повідомлень.
    - `compare`: Порівняння різних варіантів перекладу для одного файлу.
    - `translated` / `untranslated`: Фільтрація повідомлень за станом перекладу.
    - `with-context` / `regular` / `plural`: Фільтрація за типом повідомлення.
    - `with-word` / `with-wordstr`: Пошук повідомлень за входженням підрядка.
    - `with-unequal-linebreaks`: Валідація кількості переносів рядків.
    - `check-symbols`: Валідація спеціальних символів.
- **AI Інтеграція** (WIP):
    - `translate`: Переклад PO файлу за допомогою AI (Work In Progress).
    - `review`: Рецензування перекладів за допомогою AI (Work In Progress).
    - `filter`: Фільтрація та оцінка повідомлень AI (Work In Progress).
    - **Абстракція бекенду**: Підтримка різних CLI-інструментів (`aichat`, `ollama`, `llm` тощо) через параметр `--ai-command`.

## Структури даних (Data Structures)
Основною структурою є `Parser`, який конфігурується кількістю форм множини (`number_of_plural_cases`).
Повідомлення ймовірно представлені внутрішньою структурою, що містить:
- `msgid`
- `msgstr` (або список `msgstr` для множини)
- `msgctxt` (опціонально)
- Коментарі

### Розширені функції
### Словники (Dictionaries)
**Мета**: Покращити якість перекладу AI та забезпечити консистентність термінології.

**Формат словника**:
- Файл у форматі **TSV** (Tab-Separated Values).
- **Колонка 1 (Ключ)**: Англійське слово або словосполучення.
- **Колонка 2 (Значення)**: Пропонований переклад.

**Використання**:
- Підтримка **декількох** словників одночасно.
- Додаються через аргумент командного рядка `-d` або `--dictionary`.
  *Приклад*: `po-tools translate file.po -d dict1.tsv --dictionary dict2.tsv`

**Логіка роботи**:
1. Програма сканує `msgid` (оригінальне повідомлення).
2. Шукає входження слів або словосполучень зі словника.
    - **Fuzzy matching (частковий)**: Підримка пошуку слів із суфіксом `-s` (множина) для іменників. (Наприклад, якщо в словнику є "bug", то "bugs" теж має знайтися).
3. Якщо знайдено збіг, до повідомлення додається контекстний коментар (або передається в prompt AI).

**Формат виводу (контексту)**:
Коментар додається у форматі:
`# KEY_WORD - TRANSLATION`

Цей блок обгортається в тег `<dictionary></dictionary>`.

*Приклад*:
Якщо в словнику є запис: `bug\tлатка`
І текст: `Fixing bugs in the system.`
То згенерований контекст: `# bug - латка`
