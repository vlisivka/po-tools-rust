# План: Абстракція AI-бекенду для перекладу

Цей план описує впровадження гнучкого механізму виклику AI-команд, що дозволить користувачам вибирати між різними інструментами (aichat, ollama, llm тощо) та значно спростить тестування.

## Огляд

Наразі виклик AI-інструментів зав'язаний на використання `aichat`. Мета цього етапу — впровадити структуру `AiBackend`, яка інкапсулює логіку виклику зовнішнього процесу та надає можливість:
1. Використовувати довільну CLI-команду через аргумент `--ai-command`.
2. Використовувати вбудовані mock-відповіді в тестах, відмовившись від складних shell-скриптів.
3. Забезпечити однакову поведінку для всіх команд, що використовують ШІ (`translate`, `review`, `filter`).

## Зміни в архітектурі

### 1. `src/util.rs`
Додати структуру `AiBackend` та методи для її створення та виконання:
- `AiBackend::new(command, args)`
- `AiBackend::with_aichat_defaults(model, role, rag)`
- `AiBackend::from_command_line(cmd_str)`
- `AiBackend::execute(prompt) -> Result<String>`
- `AiBackend::mock(response)` (для тестів)

### 2. Сигнатури команд
Оновити внутрішні конфігураційні структури (наприклад, `TranslateConfig`) для використання `AiBackend` замість окремих полів `aichat_command` та `aichat_options`.

## Критерії успіху
1. Всі існуючі тести проходять (після рефакторингу на mock).
2. Можна запустити `po-tools translate --ai-command "cat" file.po`, і програма виведе вхідний промпт як результат (імітація перекладу).
3. Можна використовувати `ollama run model` як команду перекладу безпосередньо.
4. Код став чистішим завдяки видаленню дубльованого парсингу команд aichat у різних модулях.
